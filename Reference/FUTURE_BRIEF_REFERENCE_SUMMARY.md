# Future Pilot Brief Reference Summary

Purpose: use this as the working index for turning the `reference/` folder into future Gold Standard Pilot Brief content. The source references are still the authority; this file is a practical map for what to pull into each brief.

## Folder Roles

### `reference/777/`
Use for aircraft-specific 777 review, limitations, memory items, flows, systems study prompts, and captain technique.

Best sources:
- `777 ALL Training Notes.md`: personal quick notes, recurring technique reminders, windshear/turbulence/approach/taxi/takeoff cues.
- `Jim Notes Jan 2025.pdf`: strong source for flows, model IDs, FMC/ACARS setup, normal flow reminders, review prompts, and 777 limitation-style review.
- `777 MEMORY OCT22.pdf`: memory items and key limitations; use sparingly as a review box, not as a substitute for the manual.
- `777-aircraft-systems-study-guide.pdf`: system-of-the-day question bank.
- `777 Flight Manual.pdf`, `777_FRM.pdf`, `FOM_QRG.pdf`: searchable reference library for exact manual lookups.
- `777 Qualification Course JAN 2.pdf`: training scenario themes, systems lesson topics, qualification standards, and LOFT-style discussion prompts.

### `reference/Briefings/`
Use for the generated brief structure and flow.

Brief sections to drive:
- Flight Attendant brief.
- Captain departure brief.
- Pilot Flying departure brief.
- Arrival brief.
- Relief pilot change brief.
- Required dispatch report reminders.
- Turbulence action guide.
- International pilot/dispatcher briefing.

Note: several PDFs in this folder are image/screenshot PDFs, so direct text extraction is weak. The same core briefing structure is searchable in `reference/777/FOM_QRG.pdf`.

### `reference/Company/`
Use mostly for company-context searches, not routine flight brief clutter.

Best use:
- `777 CA June 2026.pdf`: company/current-aircraft or captain/admin reference when relevant.
- `UAL Base Guide.pdf`, `Reserve Guide.pdf`, `UPA23...pdf`, `Category Summary...pdf`, `Integrated Seniority List.pdf`: useful for schedule, reserve, bidding, base, or contract questions. Avoid pulling these into ordinary flight briefs unless the brief has a crew/schedule/admin purpose.

## High-Value Brief Content To Use Automatically

### Top-of-Brief Threat Lens
Each new flight should start with a short threat list from uploaded OFP/trip kit plus reference reminders:
- Personal, environmental, technical threat lens.
- Weather/turbulence, windshear, runway condition, NOTAM/taxi, MEL/CDL, fuel, ETOPS/oceanic, arrival complexity.
- Keep the top list operational and short. Use references to sharpen the wording, not to flood the page.

### Flight Attendant Brief
Pull from QRG/briefing references:
- Crew introductions.
- FAM/FFDO/LEO/jumpseat.
- Relief pilot considerations.
- Flight time and short taxi.
- Weather/turbulence.
- Cabin maintenance items.
- Channel 9 if installed/applicable.
- TEST for non-normal/emergency: Type, Evacuation, Special instructions, Time to landing.

Suggested app output:
- A short FA brief card.
- A turbulence-specific line when OFP/trip kit includes moderate or greater turbulence.
- A TEST reminder in the cabin/PA page.

### Captain Departure Brief
Pull from QRG and personal notes:
- Fit for duty.
- PED/EFB cellular data off.
- Pilot duties and currency.
- Current/correct pubs.
- Release verification.
- Fuel plan.
- Maintenance status.
- NOTAMs, ops alerts, pilot bulletins.
- Taxi plan, hot spots, 10-7 notes.
- RTO and evacuation.
- Air return/takeoff alternate.

Suggested app output:
- Departure brief card should combine actual release data with the above checklist.
- If trip kit has construction/closures, put taxi/10-7 items high.
- If gusty/wet/windshear conditions exist, highlight new takeoff data and max/full thrust consideration.

### Pilot Flying Brief
Pull from QRG:
- PDC/clearance.
- Departure review.
- Terrain/obstacles.
- Transition altitude.
- Weather/windshear.
- Planned takeoff data.
- Takeoff profile.
- Engine failure profile.
- Display/automation modes.
- Other threats/mitigations.

Suggested app output:
- PF brief should be runway/SID-specific.
- Include a compact engine-failure profile prompt and automation mode plan.
- If the flight includes Class II/oceanic, add FMC route verification and waypoint verification reminders.

### Arrival Brief
Pull from QRG and stabilized approach notes:
- FMC/FMGC programming.
- ATIS, NOTAMs, MEL.
- Landing performance.
- Slippery runway.
- Arrival review and transition level.
- Terrain/obstacles.
- If night or IMC: briefing cards, strip/plate.
- Go-around profile.
- Windshear/PWS.
- Runway exit/taxi/hot spot.
- Other threats/mitigations.

Suggested app output:
- Arrival page should merge STAR/runway/ATIS/landing fuel/NOTAMs with stabilized approach gates.
- Keep go-around and runway-exit plan visible.

### Relief Pilot Change Brief
Pull from QRG:
- Position/flight level.
- Clearance, impending clearance, route changes.
- CPDLC/ADS status.
- Radio frequencies.
- Enroute weather/turbulence.
- Status of diversion/landing airports.
- ATC and Ops Control status.
- Redispatch status/weather reports.
- Autopilot modes selected.
- Aircraft and cabin status.
- Mach/cost index.
- Routing and altitude strategy.
- Fuel score / nav score.
- Acting captain designation if captain absent.

Suggested app output:
- For long-haul/augmented trips, include a one-panel relief handoff page.
- Pre-fill with current phase/timeline items when available.

### International Pilot/Dispatcher Brief
Use for flag/ETOPS/oceanic flights:
- Maintenance status: MELs/CDLs, dispatch items, ETOPS.
- Payload and fuel: planned CI, payload issues, reserves including 5%, 10%, redispatch.
- Departure: security, weather, NOTAMs, pilot bulletins, crossing times/slot/delay/CFMU, track message, route, Class I/Class II.
- Enroute: ETOPS/alternates, driftdown/decompression ETP vs redispatch point, turbulence, volcanic ash, temperature, solar activity, overflight clearance numbers.
- Arrival: ATC, curfews, arrival rates, connection issues, arrival weather, NOTAMs.

Suggested app output:
- Add a dedicated Dispatcher/International box when route is oceanic/ETOPS/flag.
- Use the OFP to populate ETOPS alternates, critical point, fuel gates, and route class.

### Required Dispatch Reports
Use as a decision/reminder card:
- More than 15 minute increase in flight time.
- Plus/minus 5 minute change from a reported ETA.
- 4000 ft cruise altitude change.
- More than 100 NM lateral deviation.
- Flight deck fuel quantity indicator fails.
- All contingency fuel will be used.
- Significantly different weather or moderate/severe turbulence.
- Enroute or arrival holding.
- Unplanned or sustained use of deicing/anti-icing.
- Assigned CDR.
- CDR cases where fuel requirement, driftdown, CCO timing, revised release, or route offset matters.

Suggested app output:
- Keep this in the timeline/ETOPS page or Captain's Corner.
- Trigger emphasis if actual flight includes reroute, altitude change, holding, fuel pressure, or turbulence.

### Turbulence Action Guide
Use for cabin and PA cards:
- Forecast/impending moderate or greater: "Flight attendants take your jumpseats."
- When improved and safe to resume: "Flight attendants check in."
- Unexpected moderate or greater: "Flight attendants be seated immediately, be seated immediately."
- If improved but duties not safe: return to jumpseats.
- If ceased and safe: check in; lead FA reports cabin status.

Suggested app output:
- If OFP/trip kit includes moderate/severe turbulence, show a turbulence timeline with PA wording and FA response.
- Tie it to EET/waypoints from dispatch remarks.

### Stabilized Approach / Go-Around
Use in every arrival page:
- By 1500 ft AGL/RA or FAF, whichever later: gear down, airspeed <= 180 kt.
- By 1000 ft: final landing configuration, landing/final descent checklist, target speed +15/-5, lateral/vertical profile, max vertical-speed criteria, PM deviations.
- At/below 500 ft: except momentary deviations, PM calls go-around and reason.
- 777 uses radio altitude for gate references.

Personal notes add:
- PM call flow: 1000, 500 stable/unstable.
- Landing: maintain target to flare; flare around 30 ft; reverse promptly after touchdown; do not go around after reverse activation.

Suggested app output:
- Use as a reusable stabilized approach card.
- Highlight runway, landing distance, braking action, windshear, and taxi exit plan beside it.

### Windshear / Microburst
Use from training notes and memory/reference cards:
- If controller broadcasts microburst/microburst alert/shears of 30 kt or greater for affected runway, no takeoff initiation or descent below 1000 ft AGL for that runway.
- Precaution strategy: max/full thrust, longest suitable runway, consider flaps 20 if performance allows, delete reduced thrust where required, use actual/current winds and current ATIS.
- Escape reminder: TO/GA, maximum thrust, stow speedbrakes, wings level, rotate/pitch toward 15 degrees, follow FD if available, do not change flap/gear/trim until safe.

Suggested app output:
- Trigger on windshear/microburst/gusty/wet runway words in ATIS/NOTAM/weather/trip kit.
- Place in departure page and arrival page when relevant.

### 777 Limitations / Memory Review
Use as a rotating review, not all at once:
- Max operating altitude: 43,100 ft.
- Max takeoff/landing tailwind component: 15 kt.
- Severe turbulence speed: at/above 25,000 ft 280 KIAS/.82 Mach whichever lower; below 25,000 ft 270 KIAS; maintain at least 15 kt above min maneuver speed when below .82 Mach.
- Engine anti-ice AUTO/ON when TAT/OAT 10 C or below and icing conditions exist/anticipated, with manual exceptions per FM.
- Autopilot minimum engagement height after takeoff/missed approach: 200 ft AGL.
- Non-ILS: autopilot disengage before more than 50 ft below MDA/DA unless in go-around.
- Do not use FLCH on final approach below 1000 ft AFE.
- Without LAND 2/LAND 3, autopilot disengage below 200 ft AGL.
- HF: do not transmit during fueling.
- Engine oil minimum before start: 18 quarts.
- Cabin altitude memory item: oxygen masks, establish pilot communications.
- Stabilizer memory item: both STAB cutout switches CUTOUT, do not exceed current airspeed.

Suggested app output:
- One "System / Limitation of the Day" rotates each trip.
- Tie the system review to trip threats when possible: fuel for long-haul, anti-ice for winter, hydraulics/flight controls for abnormal scenarios, communications for oceanic.

### 777 Systems Study Guide
Use as a question source:
- General/airplane.
- Air systems.
- Anti-ice/rain.
- Autoflight.
- Communications.
- Electrical.
- Engines/APU.
- Fire protection.
- Flight controls.
- Instruments/displays.
- FMS/navigation.
- Fuel.
- Hydraulics.
- Landing gear.
- Warning systems.

Suggested app output:
- Rotate systems every generated brief.
- Ask one or two brief questions and include compact answer/reminder.
- Avoid repeating the same system on consecutive trips unless selected by threat context.

### Captain Flow / Normal Technique
Use from Jim Notes and training notes:
- Initial setup: ECL database/FMC IDENT, POS INIT, ADIRU, route uplink/loading, independent route verification, Class II waypoint verification, winds entry, runway/SID, PERF INIT, takeoff data.
- Before push: cabin ready, doors closed/armed, flight deck door secure; hydraulics, fuel pumps, doors, fuel checked, transponder, beacon, seat belts.
- After start: flaps set, electrical transfer verify, APU off, engine anti-ice auto/on, recall checked, audio set; consider delaying flight control check until deicing complete.
- Before takeoff: performance/takeoff data entered, trim/MCP set, radar/terrain as required, EFB airplane mode, departure announcement, transponder TA/RA, recall checked, checklist before crossing hold-short.
- Taxi: max taxi speeds around 30 kt straight, 8-10 kt turns; avoid riding brakes; breakaway thrust limits by model; nose sits ahead of nose gear.
- Descent/arrival: autobrakes, reference speeds, FMC/radios, altimeters, exterior lights.

Suggested app output:
- Keep these as a Captain Flow page, especially for practice briefs.
- Use only compact reminders in operational briefs unless user asks for training mode.

### ETOPS / Oceanic / ORCA
Use for oceanic or remote segments:
- Class II/oceanic checks: clearance, FMS route/waypoint verification, HF/SELCAL/CPDLC/ADS as applicable.
- ETOPS/alternates, critical point, driftdown/decompression ETP, redispatch point.
- Driftdown/diversion reminder: offset 5 NM left/right using LNAV, minimize descent until offset, decide whether descending below FL290, then proceed according to contingency/diversion plan; maintain 500 ft vertical offset as appropriate; obtain/follow ATC clearance as able.
- ORCA action groups: engine failure, rapid decompression, weather deviation, cargo fire, medical/security.

Suggested app output:
- ETOPS map/timeline page should show entry/exit, CP, alternates, fuel gates, and ORCA quick action.
- Add driftdown/offset box when ETOPS/oceanic is present.

## What Should Not Be Auto-Inserted

- Personal non-flight notes in `777 ALL Training Notes.md` should be ignored unless moved to a dedicated personal notes file.
- Company contract/seniority/base documents should not appear in ordinary flight briefs.
- Full Flight Manual/FRM content should be searchable on demand, not dumped into the brief.
- Attachment images should be used only after they are named/tagged or manually selected; many are useful screenshots but filenames do not describe their content.

## Recommended Future App Behavior

1. Build a small reference index from this folder when the app starts or when "Refresh resources" is pressed.
2. Search by phase: departure, enroute, ETOPS, arrival, cabin, systems, dispatch reports.
3. Search by trigger words from uploaded flight plan/trip kit: ETOPS, redispatch, CDR, turbulence, windshear, microburst, volcanic ash, solar, anti-ice, MEL/CDL, runway closure, braking action, hold, altitude change.
4. Add only the highest-value 3-6 reference reminders to each brief page.
5. Rotate one 777 system and one limitation/memory reminder per trip.
6. Preserve source labels internally so a future page can show "from QRG", "from Jim Notes", or "from Memory Items" without cluttering the pilot-facing brief.
